File Naming Convention

Standard Format

[topic]_[type]_[status].html

Topic Categories:

  • nuclear_speckles - Nuclear speckle research
  • chromatin_dynamics - Chromatin organization
  • dna_repair - DNA repair mechanisms
  • epigenetic_inheritance - Epigenetic studies
  • nuclear_organization - Nuclear architecture

Review Types:

  • comprehensive_review - Full detailed analysis
  • critical_review - Critical evaluation
  • methods_review - Methodological focus
  • update_review - Recent developments

Examples

✓ GOOD
nuclear_speckles_comprehensive_review.html
✓ GOOD
dna_repair_pathway_crosstalk_complete.html
✗ AVOID
NuclearSpecklesReview.html
✗ AVOID
review_1_final_updated.html

    Header Section

    Title, author info, publication date, tags

  2. 2

    200-300 word overview of key findings

  3. 3
    Table of Contents

    Navigational aid for long reviews

  4. 4
    Main Content Sections

    Logical progression with clear headings

  5. 5
    Figures & Illustrations

    Scientific diagrams with proper captions

  6. 6
    References & Citations

    Complete bibliographic information

Writing Guidelines

Scientific Accuracy

  • • Cite peer-reviewed sources
  • • Use precise scientific terminology
  • • Include methodology details
  • • Acknowledge limitations and controversies

Readability

  • • Define technical terms on first use
  • • Use active voice when possible
  • • Break up long paragraphs
  • • Include visual elements to aid comprehension

Consistency

  • • Follow established terminology
  • • Maintain consistent formatting
  • • Use standardized abbreviations
  • • Apply uniform citation style

Performance Guidelines

Images

  • • Use WebP format when possible
  • • Include alt text for accessibility
  • • Optimize file sizes
  • • Use responsive images

CSS

  • • Minimize HTTP requests
  • • Use CSS Grid/Flexbox
  • • Avoid inline styles
  • • Compress production CSS

JavaScript

  • • Load scripts asynchronously
  • • Use modern ES6+ features
  • • Minimize DOM manipulation
  • • Implement lazy loading

Accessibility Standards

WCAG 2.1 AA Compliance

Visual

  • • Minimum 4.5:1 contrast ratio
  • • Text resizable up to 200%
  • • No information conveyed by color alone
  • • Focus indicators clearly visible

Keyboard

  • • All functionality keyboard accessible
  • • Logical tab order
  • • Skip navigation links
  • • No keyboard traps

Screen Readers

  • • Semantic HTML structure
  • • Proper heading hierarchy
  • • Alt text for images
  • • ARIA labels where needed

Testing Tools

  • • Lighthouse accessibility audit
