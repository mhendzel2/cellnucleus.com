Content Guidelines

Writing Standards

  • Use clear, concise scientific language
  • Maintain academic tone while ensuring accessibility
  • Include proper citations and references
  • Use active voice when possible
  • Define technical terms on first use
  • Include visual elements to support complex concepts

Review Structure

  1. Abstract/Executive Summary
  2. Introduction & Background
  3. Main Content Sections
  4. Discussion & Implications
  5. Conclusions
  6. References & Further Reading

Content Quality Checklist

Scientific accuracy verified
Sources properly cited
Visual elements integrated
Technical terms defined
Consistent formatting applied
Peer review completed

Technical Specifications

HTML Standards

  • HTML5 semantic elements
  • Proper heading hierarchy (h1-h6)
  • Alt text for all images
  • ARIA labels for accessibility
  • Valid HTML markup
  • Meta tags for SEO

CSS Framework

<link href="https://cdn.jsdelivr.net/npm/tailwindcss@2.2.19/dist/tailwind.min.css" rel="stylesheet">

Performance Requirements

  • Page load time < 3 seconds
  • Mobile-first responsive design
  • Optimized images (WebP when possible)
  • Minified CSS and JavaScript
  • CDN usage for external libraries
  • Lazy loading for images

Browser Support

  • Chrome (latest 2 versions)
  • Firefox (latest 2 versions)
  • Safari (latest 2 versions)
  • Edge (latest 2 versions)

File Structure & Naming

Directory Structure

cellnucleus.com/ ├── nuclear_biology_reviews/ │ ├── [topic]_comprehensive_review.html │ ├── [topic]_critical_review.html │ └── [topic]_enhanced_review.html ├── assets/ │ ├── images/ │ ├── css/ │ └── js/ ├── templates/ ├── docs/ └── index.html

Naming Convention

Review Pages:
[topic]_[type]_review.html
e.g., chromatin_comprehensive_review.html
Assets:
[page-name]_[element].ext
e.g., nuclear_speckles_diagram.svg
Classes:
.cellnucleus-[element]
e.g., .cellnucleus-review-card

Best Practices & Quality Assurance

Development Workflow

  1. Content planning and research
  2. Draft creation using templates
  3. Visual design implementation
  4. Content review and fact-checking
  5. Technical validation
  6. Accessibility testing
  7. Performance optimization
  8. Final review and publication

Quality Assurance

✓ DO
  • Use semantic HTML
  • Include alt text for images
  • Test on multiple devices
  • Validate HTML markup
  • Optimize for performance
✗ DON'T
  • Use inline styles
  • Ignore accessibility
  • Skip testing phase
  • Use deprecated HTML
  • Neglect mobile design

Implementation Tips

  • Start with templates: Use established templates for consistency
  • Mobile-first approach: Design for mobile devices first, then scale up
  • Performance monitoring: Regularly check page load times and optimize
  • User feedback: Collect and incorporate user feedback for improvements
  • Version control: Maintain versions of all pages for rollback capability
